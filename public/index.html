CloudSH

被控主机无需公网 IP，无需开放端口，通过云中继转发纯文本命令。

一个 HTTP API 驱动的 SSH 替代方案——Agent 主动外连，穿透任意防火墙。适合管理内网机器、IoT 设备、临时 VPS。

你
cloudsh exec
→
云 VPS
Server :8080
←
被控主机
Agent 轮询

快速开始

1. 云 VPS 启动 Server
零依赖，单个 2MB 二进制
$ CLOUDSH_PORT=8080 cloudsh-server
2. 本地注册获取凭证
$ cloudsh -s http://VPS_IP:8080 register
client_id:  a1b2c3d4
password:   xxxxxxxxxxxxxxxx
3. 被控主机启动 Agent
$ CLOUDSH_SERVER=http://VPS_IP:8080 \
  CLOUDSH_CLIENT_ID=a1b2c3d4 \
  CLOUDSH_PASSWORD=*** \
  cloudsh-agent
4. 执行命令
$ cloudsh -s http://VPS_IP:8080 exec whoami
root
$ cloudsh -s http://VPS_IP:8080 exec "ls -la /var/log"

安装

一键编译 + systemd 服务
需要 Rust 1.80+，脚本自动安装
$ git clone https://github.com/yxpil/CloudSH
$ cd CloudSH
$ bash install.sh          # 全编译
$ bash install.sh server   # 或仅 Server
$ bash install.sh agent    # 或仅 Agent
$ bash install.sh client   # 或仅 Client
systemd 自动启动
$ sudo systemctl enable --now cloudsh-server
$ sudo systemctl enable --now cloudsh-agent   # 先编辑凭证

安全

纯文本转发
拒绝二进制和控制字符（\x00 \x1b），只过纯文本命令
常量时间密码比较
防时序攻击，密码长度泄露也无用
无 SSH 凭证存储
Server 不存目标机 SSH 密码，Agent 本地执行
